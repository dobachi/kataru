# ロードマップ

アジャイル／インクリメンタルに進める。各段階で「動くもの」を出すことを優先する。

## マイルストーン

| 段階 | ゴール | 主な成果物 | 状態 |
|---|---|---|---|
| **S0** | 土台 | リポジトリ・Godotプロジェクト雛形・コンセプト文書 | ✅ 着手中 |
| **S1** | 古風2Dの土台 | タイルマップ表示 + プレイヤーの4方向移動 | ⬜ |
| **S2** | データ駆動 | シナリオ記法v0 + 変換ツール + マップを `data/` から読み込み | ⬜ |
| **S3** | 物語の最小形 | NPC配置 + markdown からの会話表示 | ⬜ |
| **S4** | AI育成 | 種テキスト→シナリオ展開の Claude スキル | ⬜ |
| S5以降 | 肉付け | クエスト / アイテム / インベントリ / 戦闘 / セーブ・ロード | ⬜ |

## 最初のマイルストーン

**「まず歩いてNPCと話せる」（S3完了時点）** を最初の目に見える達成点とする。
ここまでで、データ駆動・markdown起点・古風2Dという中核コンセプトがひと通り動く。

## 各段階のメモ

### S1: 古風2Dの土台
- タイルマップを1枚、ハードコードで表示
- プレイヤースプライトをグリッド単位で移動（古風RPG風の歩行）
- カメラ追従
- まだ markdown とは繋がない。描画と操作の感触を作る

### S2: データ駆動
- `scenario/maps/*.md` の記法v0を確定（[scenario-schema.md](scenario-schema.md) を更新）
- `tools/` で markdown → `data/*.json`
- S1のマップを「データから読む」形に置き換える

### S3: 物語の最小形
- `scenario/npcs/*.md` の記法
- マップ上にNPCを配置し、話しかけると会話ウィンドウ表示
- 会話の分岐は最小限（まず一本道）

### S4: AI育成
- `.claude/skills/` に、種テキストから `scenario/` を展開するスキルを実装
- 記法の検証（lint）もスキル or tools 側に持たせる
