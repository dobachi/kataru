# ロードマップ

アジャイル／インクリメンタルに進める。各段階で「動くもの」を出すことを優先する。

開発は **2つのトラック** を並行させる:

- **トラックA：ゲームエンジン（実行時）** — `src/` 。マップ描画・移動・会話・戦闘など、遊べる本体。
- **トラックB：制作支援（開発時）** — `tools/`（決定的な変換・検証）と `.claude/skills/`（AIによる生成・育成）。シナリオを書く／育てる人とAIを支える。詳細は [authoring-tools.md](authoring-tools.md)。

「markdownを源にAIで育てる」というコンセプトは、トラックBが充実して初めて回り始める。エンジンと制作支援を両輪で育てる。

## 最初の目に見える達成点

**「まず歩いてNPCと話せる」（S3完了時点）**。ここでデータ駆動・markdown起点・古風2Dの中核が動く。

## トラックA：各段階のメモ

### S1: 古風2Dの土台
- タイルマップを1枚表示（アセットが無くても進めるよう、コードで生成した簡易タイルでも可）
- プレイヤースプライトをグリッド単位で移動／カメラ追従
- まだ markdown とは繋がない。描画と操作の感触を作る

### S2: データ駆動
- `scenario/maps/*.md` の記法v0を確定（[scenario-schema.md](scenario-schema.md) を更新）
- S1のマップを「`data/` から読む」形に置き換える

### S3: 物語の最小形
- `scenario/npcs/*.md` を読み、マップにNPC配置 → 話しかけると会話ウィンドウ
- 会話分岐は最小限（まず一本道）

## トラックB：制作支援（要点）

制作支援の機能バックログは [authoring-tools.md](authoring-tools.md) に集約する。主な柱:

- **マップ補助**（S2）: lint・preview・scaffold。ASCIIを源に保ったまま手書き／AI生成を支える
- **md→json 変換**（S2）: ビルド時にシナリオをGodot向けデータへ
- **scenario-grow スキル**（S4）: 種テキストから世界観・マップ・NPC・会話を展開
- **初期セット作成ウィザード**（S4）: 対話で新規シナリオ一式（タイトル/世界観/開始マップ/最初のNPC・クエスト）を生成
- **整合チェック**（S5以降）: 参照切れ・設定矛盾の検出（決定的lint＋AIによる lore チェック）
