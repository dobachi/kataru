# 制作支援ツール（開発時）

「markdown を源に、AI で育てる」というコンセプトは、人とAIが協調してシナリオを作るための
支援があって初めて回り始める。ここではその制作支援（トラックB）を集約する。

## 2系統

| 系統 | 置き場所 | 性質 | 例 |
|---|---|---|---|
| **決定的ツール** | `tools/` | 再現可能・テスト可能。入力が同じなら出力も同じ | 変換・lint・scaffold・preview |
| **AIスキル** | `.claude/skills/` | 対話的・生成的。創作を膨らませる | 育成・ウィザード・整合チェック |

役割分担の原則：**「正しさの検証」は決定的ツール**、**「内容の創出」はAIスキル**。
両者は協調する（例：ウィザードがAIで内容を作り、scaffold が決定的に雛形を置く）。

## 初期セット作成ウィザード（詳細）

新しいシナリオを始める人が、白紙から迷わないための入口。

- **入力（対話）**: タイトル / 世界観の一言 / 雰囲気 / 開始マップの規模 / 最初のNPC・クエストの有無
- **出力**: `scenario/world.md`、開始マップ1枚、NPC1人、（任意で）最初のクエスト一式
- **構成**: AIスキルが内容を創出し、`tools/` の scaffold が記法に沿った骨格を決定的に配置する
- **検証**: 生成直後に lint を通し、壊れた雛形を出さない

## scenario-grow（詳細）

育成のループ：
1. 作者が `scenario/` に短い種を書く（または既存を指定）
2. スキルが記法に沿って展開・補完（世界観→地名→マップ→NPC→会話…）
3. lint で整合を確認
4. 人がレビューして markdown を確定

一度に作り込みすぎず、差分でレビューできる粒度を保つことを重視する。

## 設計上の指針

- **源を壊さない**: ツールは `scenario/*.md`（源）を勝手に上書きせず、生成・提案・検証に徹する（明示時のみ書き込み）
- **AIなしでも回る**: 決定的ツールだけでビルド・検証が完結する。AIスキルは上乗せの加速装置
- **小さく出す**: 各ツールも MVP から。まずマップ lint と変換、次に preview/scaffold
